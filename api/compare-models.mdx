## Description

The `main()` function in `compare_models.py` loads all saved model metrics from the `results/` directory, compares them side-by-side in a formatted table, and identifies the best-performing model based on Test R² score.

## How It Works

<Expandable title="Step-by-step process">
  
  1. **Check Results Directory** - Verifies that `results/` directory exists
  2. **Load Metrics Files** - Reads all `metrics_*.json` files from the directory
  3. **Extract Key Metrics** - Parses JSON files to extract:
     - Model Name
     - Train R²
     - Test R²
     - CV R² Mean
     - Train RMSE
     - Test RMSE
  4. **Create DataFrame** - Organizes metrics into a pandas DataFrame
  5. **Sort by Test R²** - Ranks models in descending order (higher is better)
  6. **Display Table** - Prints formatted comparison table
  7. **Identify Winner** - Highlights the best model with trophy emoji
  
</Expandable>

## Input Requirements

<ParamField path="results/" type="directory" required>
  A directory containing JSON files with model metrics. Each file must:
  
  - Follow naming convention: `metrics_*.json` (e.g., `metrics_linear_regression.json`)
  - Contain metrics generated by `evaluate_model()` function
  - Include at minimum: `model_name`, `train_r2`, `test_r2`, `train_rmse`, `test_rmse`
</ParamField>

## Ranking Criteria

**Primary Metric: Test R² (Descending)**

Models are ranked by their Test R² score because:

- **Test R²** measures performance on unseen data, indicating real-world generalization
- Higher values (closer to 1.0) indicate better predictive accuracy
- More reliable than Train R² for assessing true model quality
- Prevents selection of overfitted models that only perform well on training data

<Expandable title="Understanding R² scores">
  
  - **R² = 1.0**: Perfect predictions
  - **R² = 0.85+**: Excellent model (explains 85%+ of variance)
  - **R² = 0.70-0.84**: Good model (explains 70-84% of variance)
  - **R² = 0.50-0.69**: Moderate model (explains 50-69% of variance)
  - **R² < 0.50**: Poor model (explains less than half the variance)
  - **R² < 0**: Model performs worse than simply predicting the mean
  
</Expandable>

## Complete Workflow Example

```python
import json
from pathlib import Path
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression
from sklearn.tree import DecisionTreeRegressor
import pandas as pd

# 1. Prepare data
df = pd.read_csv('house_prices.csv')
X = df[['area', 'bedrooms', 'bathrooms', 'age']]
y = df['price']
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 2. Create results directory
Path('results').mkdir(exist_ok=True)

# 3. Train and save metrics for multiple models
models = [
    ('Linear Regression', LinearRegression()),
    ('Decision Tree', DecisionTreeRegressor(max_depth=10)),
]

for name, model in models:
    # Train model
    model.fit(X_train, y_train)
    
    # Evaluate and get metrics
    metrics, _ = evaluate_model(
        model, X_train, X_test, y_train, y_test, name
    )
    
    # Save metrics to JSON
    filename = f"results/metrics_{name.lower().replace(' ', '_')}.json"
    with open(filename, 'w') as f:
        json.dump(metrics, f, indent=2)
    
    print(f"Saved metrics for {name}")

# 4. Compare all models
print("\n" + "="*80)
print("COMPARING ALL MODELS")
print("="*80 + "\n")

from compare_models import main
main()
```

## Error Handling

<Expandable title="Common errors and solutions">
  
  ### Error: "Results directory not found"
  
  **Cause:** The `results/` directory doesn't exist
  
  **Solution:** Create the directory and save model metrics first:
  ```python
  from pathlib import Path
  Path('results').mkdir(exist_ok=True)
  ```
  
  ### Error: "No metrics JSON files found"
  
  **Cause:** No files matching `metrics_*.json` pattern in `results/` directory
  
  **Solution:** Ensure you've trained models and saved their metrics:
  ```python
  metrics, _ = evaluate_model(model, X_train, X_test, y_train, y_test, "Model Name")
  with open('results/metrics_model_name.json', 'w') as f:
      json.dump(metrics, f)
  ```
  
  ### Missing Metrics Keys
  
  **Cause:** JSON files don't contain expected keys (train_r2, test_r2, etc.)
  
  **Solution:** Use `evaluate_model()` to generate properly formatted metrics
  
</Expandable>
